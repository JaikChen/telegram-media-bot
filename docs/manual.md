# 📖 Telegram Media Bot 使用手册

## 📌 核心逻辑说明
本机器人主要负责从 **源频道** 监控媒体消息，经过 **内容清洗** 后，转发到 **目标频道**。

### 1. 转发配置流程
1. **添加机器人**：将机器人添加为源频道和目标频道的管理员（至少拥有发送消息权限）。
2. **建立转发关系**：
   - 命令：`/addforward <源频道ID> <目标频道ID>`
   - 示例：`/addforward -100123456789 -100987654321`
3. **设置延迟（可选）**：
   - 默认即时转发。
   - 命令：`/setdelay <最小秒数> <最大秒数>`
   - 示例：`/setdelay 60 300` (随机 1-5 分钟后转发)

### 2. 内容清洗规则 (Rules)
每个频道可以独立设置清洗规则：
- `clean_links`: 自动删除文本中的链接（支持实体链接、Markdown、HTML、裸 URL）。
- `remove_at_prefix`: 删除 `@username` 形式的提及。
- `pangu`: 在中英文之间自动添加空格，优化排版。
- `require_spoiler`: 强制媒体必须有剧透马赛克（或包含 `#nsfw`），否则拦截。
- `keep_all`: 禁用所有清洗逻辑，原样转发。
- `strip_all_if_links`: 如果消息中包含链接，直接丢弃整条消息。

### 3. 关键词过滤
- **屏蔽词 (`/block`)**: 包含关键词的消息将被直接拦截（不转发）。
- **清洗词 (`/clean`)**: 仅删除包含关键词的行，保留其余内容。

### 4. 投票与信誉 (Karma)
- **投票按钮**: 使用 `/setvoting on` 为频道开启 👍/👎 投票。
- **信誉豁免**: 经常获得 👍 的用户会增加信誉值（Karma）。高信誉用户发送的内容将跳过清洗引擎。

## 🛠 管理员指令参考

| 分类 | 指令 | 说明 |
| :--- | :--- | :--- |
| **转发管理** | `/addforward` | 建立转发链 (源 -> 目标) |
| | `/delforward` | 解除转发链 |
| | `/listforward` | 查看当前频道的转发去向 |
| **规则设置** | `/setrule <规则名>` | 为当前频道开启规则 (再次输入关闭) |
| | `/listrules` | 查看当前频道生效的规则 |
| **词库管理** | `/addkeyword <词>` | 添加屏蔽词/清洗词 |
| | `/delkeyword <词>` | 删除关键词 |
| **系统维护** | `/repair` | 重启转发工人进程 |
| | `/pause` / `/resume` | 全局暂停/恢复转发任务 |
| | `/status` | 查看系统运行状态与队列积压 |
