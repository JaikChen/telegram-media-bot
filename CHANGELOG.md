# Changelog

All notable changes to this project will be documented in this file.

## [v3.0.0] - 2026-05-23

### Added
- **ForwardingService**: A dedicated service layer for orchestrating the forwarding worker, handling rate limits, and ensuring atomicity in large-scale operations.
- **Robust Deployment Script**: Enhanced `deploy_bot.sh` (Ultimate Edition) with environment self-checks, `git clean` for total consistency, and dependency verification.

### Improved
- **MediaRepository Batch Operations**: New `enqueue_batch` and `fetch_queue_batch` methods for high-performance, transactional queue management.
- **Reliability for Large-Scale Forwarding**: Significantly improved "once-and-for-all" forwarding reliability, preventing data loss and duplicate processing through status-based locking and atomic seen-checks.
- **Service Decoupling**: Fully removed business logic from `db.py` and `handlers/media.py`, delegating all forwarding orchestration to `ForwardingService`.

### Changed
- **Architecture**: Completed the migration to a Service/Repository pattern across the entire project.

## [v2.8.0] - 2026-05-16

### Added
- **Architectural Refactoring**: Introduced `services.py` and `repositories.py` to decouple business logic from persistence and handler layers.
- **Database Inspection Utility**: New `inspect_db.py` script for monitoring queue health and identifying duplicates.
- **Atomic Queue Management**: Added unique indexes and atomic `INSERT OR IGNORE` operations in `db.py` to prevent duplicate items in the forwarding queue.

### Fixed
- **Repository Hygiene**: Cleaned up the repository by removing tracked `__pycache__` files and other unnecessary artifacts.
- **Improved .gitignore**: Updated to a comprehensive standard to prevent future pollution of the repository.

### Optimized
- **Media Handler Decoupling**: Refactored `handlers/media.py` to use specialized services, making the code more readable and testable.

## [v2.7.0] - 2026-05-12

### Added
- **Dynamic Caption Templates**: New command `/settemplate` allows users to define powerful caption templates with placeholders like `{orig}` (original text), `{title}` (chat title), `{cid}`, `{date}`, and `{user}`.
- **Media Type Filtering**: New command `/setfilter` allows chats to restrict which media types are processed (e.g., only allow `photo` and `video`, block `sticker`).
- **Batch Forwarding**: The `forward_worker` can now process up to 10 items in a single run when the queue is backing up, significantly increasing throughput for large volume transfers.
- **Enhanced Media Group Debounce**: Implemented a more robust sliding-window aggregation for media groups (albums) to prevent data loss during network spikes or mass-forwarding.
- **Source Context Preservation**: The bot now tracks and passes chat titles and other metadata to the cleaning engine for more intelligent processing.

### Fixed
- **Media Loss Issue**: Resolved a critical issue where media items (especially in large albums or high-frequency bursts) were dropped due to race conditions or cache evictions.
- **Rate Limit Resilience**: Improved handling of Telegram's `RetryAfter` errors in the forwarding worker.
- **Atomic Operations**: Improved database interaction to ensure that items are only removed from the queue after successful processing.

### Optimized
- **Database Performance**: Added indexing and optimized schema for high-concurrency environments.
- **Memory Management**: Increased TTL and max size for album caches to handle up to 5000 concurrent media group members.
- **UX/Feedback**: Better error logging and status reporting for admin commands.
